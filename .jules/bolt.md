## 2024-06-05 - Zustand Mouse Move Re-render Bailout
**Learning:** In highly interactive components with rapid event firing (like dragging tabs tracking mouse coordinates `x` and `y`), updating Zustand state continuously can cause heavy, redundant React re-renders. Zustand uses `Object.is` for state comparison. Returning a new object `{ dragState: { ...prev, x, y, zone } }` bypasses this bailout mechanism even if the values inside are functionally identical to the previous frame's coordinates and zone, triggering unnecessary updates.
**Action:** When handling high-frequency events in Zustand, explicitly compare the incoming values to the existing state values. If they are the same (e.g., `prev.x === x && prev.y === y && prev.zone === zone`), return the exact previous `state` object. This instructs Zustand to abort the update and skip re-renders.
## 2024-06-06 - Zustand Mouse Move Full Decoupling
**Learning:** Returning exactly `state` when high-frequency variables functionally don't change is a good bailout, but avoiding storing high-frequency data (like x/y coordinates) altogether in global state is even better. We found that `dragState.x` and `dragState.y` were updated on every mouse move but only consumed imperatively via DOM. Removing them eliminated all re-renders during dragging while maintaining functionality.
**Action:** Before optimizing global state updates for high-frequency events, audit consumers to see if the state is even necessary. If DOM manipulation handles it, remove the variables from global state completely to fully decouple rendering from the event.
## 2024-06-07 - Complete State Decoupling for Drag Zones
**Learning:** Even when using React state localized to a single component (like `App.tsx`), rapid state updates during a drag event (`handleDragMove`) can cause unnecessary and expensive React re-renders, especially if the component is near the top of the tree.
**Action:** For high-frequency visual updates like toggling drop-zone overlays during drag-and-drop, completely bypass both global (Zustand) and local (React) state. Instead, render hidden `div`s with static IDs and explicitly toggle their `style.display` properties via direct DOM manipulation (`document.getElementById`). This achieves the performance goal of zero React re-renders while keeping the UI fully responsive.
