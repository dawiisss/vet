# Get Support for Vet

Thank you for using Vet! If you run into issues, need assistance, or want to connect with the community, here are the channels available to you.

---

## 🔍 Troubleshooting First Steps

Before opening a support request, try these quick troubleshooting steps:
1. **Check the Log/Console**: Open developer tools in Vet (if running in dev mode) or inspect the CLI output to check for active warnings/errors.
2. **Clear Application Cache**: If the UI is rendering incorrectly, backing up and clearing your config file (`~/.config/vet/config.json5`) or removing database cache can help resolve local state corruption.
3. **Audit Node & Native Modules**: Verify you are using Node.js v18+ and native modules like `node-pty` built successfully for your architecture.

## 💬 Communication Channels

### 1. GitHub Issues
For bug reports, technical regressions, and specific feature proposals:
* **Link**: [Submit a Bug or Feature Request](https://github.com/dawiisss/vet/issues)
* Please use the issue templates and fill out all requested details.

### 2. GitHub Discussions
For general questions, configuration assistance, theme sharing, and community chat:
* **Link**: [GitHub Discussions](https://github.com/dawiisss/vet/discussions)
* Search previous discussions to see if your question has been answered before starting a new thread.

### 3. Direct Contact
For partnerships, licensing queries, or critical security vulnerabilities:
* **Email**: [vet@route2place.com](mailto:vet@route2place.com)
* Please use email only if GitHub Issues or Discussions are not appropriate for your query.
