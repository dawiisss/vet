## 2024-05-18 - [Electron IPC Open External] Learning: External URLs passed to `shell.openExternal` must be validated against a strict protocol allowlist to prevent arbitrary code execution (e.g. `file://` or custom OS protocols). Action: Parse URLs using `new URL()` and check the `.protocol` property against an allowlist before execution.
## 2024-05-18 - [Electron WebContents Security] Learning: Unrestricted window creation and navigation in Electron windows can lead to security vulnerabilities if a renderer is compromised. Action: Always implement `setWindowOpenHandler` returning `{ action: "deny" }` with safe external fallback, and listen to `will-navigate` to block untrusted domain navigation.
## 2026-06-08 - [Strict Allowlisting with DOMPurify] Learning: When rendering SQLite FTS5 `snippet()` results (which inject `<b>` tags) via `DOMPurify` and `dangerouslySetInnerHTML`, the default configuration allows too many benign but unnecessary tags. Action: Apply strict configuration like `{ ALLOWED_TAGS: ['b'], ALLOWED_ATTR: [] }` to enforce the Principle of Least Privilege and prevent potential UI manipulation or phishing through otherwise permissive allowed tags.
